安装hexo

在命令行中输入 sudo npm install -g hexo
安装成功之后在自己想要安装Hexo的路径下新建一个文件名字随便起，比如hexo_blog。
在命令行中输入cd hexo_blog 比如我的文件路径cd /Users/huxiao-mac/Desktop/hexo_blog （*我的是在苹果电脑下,windows命令后进入一个目录可以参考http://jingyan.baidu.com/article/92255446962e52851648f403.html）然后在命令行输入hexo init 演示图如下图05.png
好啦，至此全部安装工作已经完成！其实回头看看一点也不难对吧～嘿嘿 关键是老司机认识路啊

配置blog 让blog在本地展示

继续在命令行输入hexo generate （hexo g 也可以） 这行命令的作用是生成静态页面 命令执行完毕后继续在命令行输入hexo server 启动本地服务，进行文章预览调试。如果命令执行完看见了如图06.png那么恭喜你已经成功配置了本地blog。让我来看看效果图07.png这个界面是hexo给我们的一个默认模版,不喜欢的话可以进入01.png界面所指示的主题模块选择下载喜欢的模版。

更换模版 （不想更换模版的这个模块可以不用看）

Mac下在命令后工具中切换到刚刚开始我们新建的hexo_blog目录下 cd /Users/huxiao-mac/Desktop/hexo_blog然后在主题页面选择你要下载的主图如图08.png按照08.png图上指示进入下载页面后如图09.png 按照上图中的提示复制这个主题的url。然后在命令行工具中使用git clone 把粘贴的url复制到这里 例如git clone https://github.com/shuiRong/hexo-theme-Ada等下载完成后（出现100%）表示下载完成。在hexo_blog目录下找一个themes的文件打开就能看到你下载的主题 继续在hexo_blog目录下找一个_config.yml命名的文件用记事本打开,内容类似于下图10.png 可以看到有source title subTitle descriptio author url theme等单词
source 将你刚刚复制的主题url粘贴到这个位置
title subTitle 这个是blog的标题 副标题 按个人喜好起名字就行
author 这个是blog的作者
theme 主题的名字 例如刚刚复制的主题URL是这样 https://github.com/BoizZ/hexo-theme-laughing 那么最后一个/后面的就是theme 复制粘贴过去就可以了
url 这个就是GitHub这个工具上我们存放hexo blog的位置。接下来老司机教大家怎么在GitHub创建一个这样的位置。前提是你已经按照02.png成功注册了gitHub。登录后可以看到这样的界面11.png在右上角有个 New repository（新建一个仓库）点击这个按钮后会出现这样一个网页12.png这就是我们建立存储位置的界面。然后我们只需要在repository name下方的空白处输入一个名字即可 名字的格式：名字.github.io 完成之后再点击下方的绿色按钮Create repository即可完成一个简单仓库的建立。建立成功之后将http://+你刚刚创建的仓库名字 例如https://name.github.io。将URL粘贴到刚刚打开的_config.yml 文件中URL的后面。下面有很重要的一步就是配置github的ssh。详细配置请参考这个博客一步步走就可以完成ssh配置教程http://blog.csdn.net/binyao02123202/article/details/20130891

在_config.yml的最后可以看到类似这样的代码块,如果缺少某一行你就复制粘贴过去。

将repository:后面的地址替换成你刚刚创建的仓库的URL地址，不要问我怎么复制url除非你不会用浏览器。

将本地blog推送到远程（在任何一台计算机上都可以访问）

在命令行中输入npm install hexo-deployer-git --save执行这个命令才能使用git部署 看到有绿色的done单词出现表示完成。然后输入hexo generate完成后 再输入hexo deploy等待命令完成。到此我们就成功搭建好了一个blog。而你这个blog的网址就是_config.yml文件中URL后面的链接。将这个URL复制到浏览器不出意外就可以成功看到你的blog。
